# Style Guide

## Purpose
This guide defines the default writing standards for internal documentation in this repository.

## Terminology
- Use **MUST** for mandatory requirements.
- Use **SHOULD** for recommendations.
- Use **MAY** for optional actions.
- Use **checklist** for a verification list of pass/fail items.
- Use **section reference** when citing where an issue appears.

## Required Structure
Every process document MUST include the following sections in order:
1. Title
2. Objective
3. Prerequisites
4. Procedure
5. Review Checklist
6. Revision Log

## Formatting Standards
- Use sentence case headings.
- Use numbered lists for ordered steps.
- Keep each step actionable and specific.
- Keep checklist items written as yes/no verification statements.
- Use concise, direct language.

## Example Format Standard
Examples SHOULD follow this format:
- **Context:** One-sentence setup.
- **Action:** Exact step performed.
- **Result:** Observable expected outcome.
