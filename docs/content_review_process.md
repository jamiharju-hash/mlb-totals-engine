# Content review process

## Objective
Define a repeatable process to draft, review, revise, and finalize documentation content so it meets repository standards before publication.

## Prerequisites
- Access to the repository and target document.
- Familiarity with `STYLE_GUIDE.md`.
- Ability to edit and save markdown files.

## Procedure
1. Draft content following the requirements in `STYLE_GUIDE.md`.
2. Run a checklist review that verifies:
   - Terminology consistency.
   - Example formatting compliance with the standard template.
   - Presence of all required sections.
3. If issues are found:
   1. Record each issue with a section reference.
   2. Revise the affected section.
   3. Re-run the checklist.
4. Repeat review and revision until all checklist items pass.
5. Finalize and save the document.

## Review checklist
- [ ] Is terminology consistent with the defined terms in `STYLE_GUIDE.md`?
- [ ] Do all examples follow the standard `Context/Action/Result` format?
- [ ] Are all required sections present and in the required order?
- [ ] Are all identified issues documented with a section reference before revision?
- [ ] Was the checklist re-run after each revision cycle?
- [ ] Is the final version saved only after all checks pass?
